Rollups 的原理与类型

解析以太坊Layer 2核心扩展方案，探索乐观型与零知识型Rollups的技术差异

如何在不牺牲 L1 强大信任的前提下，实现交易的大规模扩展。这就是 Layer 2 的使命，特别是 Rollups 技术。

我们将深入研究 Rollups 的核心原理，以及两种主要类型在信任模型上的精妙差异。

一、 Rollups 的核心逻辑：安全性的继承

Rollups 被认为是目前以太坊最可行的扩展方案，因为它遵循一个关键原则：将执行计算移到 L2，将数据存储和安全性保证保留在 L1。

1. 工作原理：压缩、执行与数据可用性

脱链执行 (Off-Chain Execution)： 数千笔交易在 L2 网络上快速、廉价地执行，拥有自己的 L2 EVM 或兼容环境。

数据压缩与批处理 (Batching)： L2 的一个重要实体（通常称为排序器/Sequencer）将这数千笔 L2 交易打包、压缩成一个极小的数据包。

数据回传 L1： Sequencer 只将两样东西提交给以太坊主网 (L1 的智能合约)：

压缩后的交易数据： 用于保证数据可用性 (Data Availability)，确保任何人都可以在 L1 上重建 L2 的状态。

状态根/证明： 一个密码学证明，证明 L2 执行的结果是有效的。

安全性继承： L1 上的 Rollup 合约负责验证证明和处理争议。这使得 Rollup 的安全性从根本上继承了以太坊主网的最终性。

2. 挑战与创新：信任模型的差异

Rollups 的类型主要取决于它们如何向 L1 证明 L2 交易是有效的。这导致了两种截然不同的信任模型。

二、 乐观型 Rollups (Optimistic Rollups)

顾名思义，乐观型 Rollups（如 Optimism、Arbitrum）采用“先假设正确，后验证”的哲学。

信任模型： 乐观推定 (Optimistic Assumption)。排序器提交 L2 状态时，L1 上的合约默认接受其提交的状态根是正确的，不会立即进行计算验证。

欺诈证明 (Fraud Proofs)： 为了防止排序器作恶，系统引入了一个挑战期 (Dispute Window)（通常为 7 天）。

在这 7 天内，任何人都可以在 L1 上提交欺诈证明。如果有人发现 L2 的状态转换是错误的，他们可以通过在 L1 上重跑相关的 L2 交易来证明欺诈。

如果欺诈被证明，恶意的 Sequencer 将受到经济惩罚（Slashing），并且状态回滚到正确版本。

优点： 技术实现相对简单，EVM 兼容性高。

缺点： 用户从 L2 提款到 L1 需要等待完整的 7 天挑战期（除非使用流动性中继）。

三、 零知识 Rollups (ZK-Rollups)

ZK-Rollups（如 zkSync、StarkNet）采用“先证明正确，后接受”的哲学，是当前最前沿的密码学技术。

信任模型： 数学上的确定性 (Cryptographic Certainty)。排序器（或证明者）在提交 L2 状态时，必须同时附带一个零知识证明 (Zero-Knowledge Proof)。

零知识证明： 这是一个极其复杂的密码学结构，它证明了以下事实：“我知道 L2 的交易数据，并且这些数据被正确执行了，从而得出了新的状态根，但我无需透露任何原始交易数据。”

验证机制： L1 上的合约立即验证这个零知识证明。验证是计算密集但快速的。如果证明有效，L1 立即接受这个新的 L2 状态。

优点：

即时最终性： 一旦 L1 验证了证明，L2 状态就立即被最终确定，无需等待 7 天的挑战期。

隐私性潜力： 零知识证明在未来具有保护交易内容的潜力。

缺点： 零知识证明的生成过程本身计算成本高昂且复杂，且实现 EVM 的完全兼容性（zkEVM）在技术上极具挑战性。

四、 Rollups 的未来与 L1 的进化

随着 Rollups 的成熟，以太坊主网 L1 也在进行相应的改造，以更好地支持 L2。

EIP-4844 (Proto-Danksharding)： 这是 L1 的关键升级。它引入了一种新的临时数据存储空间（称为 Blobs），专门用于降低 Rollup 向 L1 提交数据时的成本。

数据可用性抽样 (DAS - Data Availability Sampling)： 最终的 Danksharding 愿景是让 L1 通过随机抽样验证 Rollups 数据的完整性，从而允许 Rollups 在保证安全的前提下，提交更大的数据量，进一步提升 L2 的吞吐量和效率。

小结：

Rollups 通过将执行层与结算层分离，成功地扩展了以太坊。我们理解了乐观型依赖于欺诈证明和 7 天挑战期，而 ZK 型依赖于零知识证明实现即时最终性。Rollups 的发展和 L1 的适配（如 EIP-4844），共同构成了以太坊模块化扩展的宏伟蓝图。
